# Phase pipeline

Every phase follows this pipeline unless `instrucciones.md` defines a stricter one:

1. validate prerequisites
2. understand current phase and scope
3. consult official docs
4. reconcile discrepancies
5. implement
6. lint and type-check
7. run unit tests
8. run integration or regression tests if applicable
9. review architecture and contracts
10. update handoff, evidence, registry, and report
